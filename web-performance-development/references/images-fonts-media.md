# Images, fonts and media

Media is most of the web's bytes; fonts sit on the critical path to readable
text. Both are solved problems — the discipline is applying the known
playbook every time.

## Image formats (July 2026 — support stable, re-verify edge cases)

| Format | Use |
|---|---|
| AVIF | Default for photos — best compression; cross-engine since 2024 |
| WebP | Fallback / broadly-safe default; near-universal |
| JPEG | Legacy fallback only |
| PNG | Only where lossless/alpha is genuinely required (then compare WebP/AVIF lossless) |
| SVG | Icons, logos, line art — minified (SVGO), inlined when tiny; sanitise user-supplied SVG (script-capable → `secure-development`) |

Serve via `<picture>`/`srcset` negotiation or a CDN that transforms per
`Accept` header (`caching-and-cdn.md` for the `Vary` implications).

## Responsive delivery

```html
<img
  src="/img/hero-1200.avif"
  srcset="/img/hero-600.avif 600w, /img/hero-1200.avif 1200w, /img/hero-2000.avif 2000w"
  sizes="(max-width: 600px) 100vw, 1200px"
  width="1200" height="600"
  alt="Launch dashboard showing live telemetry"
  fetchpriority="high">
```

- `srcset` + `sizes` lets the browser pick the smallest sufficient file —
  an accurate `sizes` matters as much as the srcset (defaulting to `100vw`
  over-fetches on every desktop).
- **`width`/`height` always** (or CSS `aspect-ratio`): the browser reserves
  the box before the bytes arrive — this single habit removes most
  image-driven CLS.
- Generate the size ladder in the build or via an image CDN; never ship a
  2000 px original into a 400 px slot.

## The LCP image vs everything else

| | LCP / above-fold hero | Below the fold |
|---|---|---|
| Discovery | In the initial HTML as `<img>` (not CSS background, not JS-injected); `preload` if genuinely late-discovered | Parser finds it when it finds it |
| Priority | `fetchpriority="high"` | Default or `low` |
| Lazy? | **Never** (`loading="eager"`, the default) | `loading="lazy" decoding="async"` |

One rule of thumb: the first viewport's images load eagerly; everything
after lazy-loads. Native `loading="lazy"` has made IntersectionObserver
lazy-loading legacy except when you need custom thresholds or effects.

## Fonts

The playbook, in order:

1. **WOFF2 only** — every current engine; other formats are dead weight.
2. **Self-host** with `Cache-Control: immutable` on hashed URLs.
   Third-party font CSS adds a connection + a blocking round trip and, with
   consolidated HTTP caches, no cross-site cache benefit.
3. **Subset** to the character sets you serve (`unicode-range` for split
   subsets); variable fonts replace multiple weights with one file — usually
   a net win beyond two weights.
4. **Preload** the 1–2 critical faces (`as="font" type="font/woff2"
   crossorigin` — required even same-origin).
5. **`font-display`**: `swap` (fallback text immediately, swap on arrival)
   for brand-critical faces; `optional` (use it only if effectively instant,
   no swap-shift ever) where speed beats brand — the honest CLS-proof
   choice.
6. **Metric-adjust the fallback** so the swap doesn't shift layout:

```css
@font-face {
  font-family: "Display";
  src: url("/fonts/display.woff2") format("woff2");
  font-display: swap;
}
@font-face {
  font-family: "Display-fallback";
  src: local("Arial");
  size-adjust: 105%;       /* generated by fallback-tuning tooling */
  ascent-override: 92%;
  descent-override: 24%;
}
h1 { font-family: "Display", "Display-fallback", sans-serif; }
```

Font *choice* and typographic craft → `frontend-development`; this file owns
delivery.

## Video and animated media

- **Animated GIF is banned** — a muted looping MP4/WebM is an order of
  magnitude smaller: `<video autoplay muted loop playsinline>`.
- `preload="none"` (or `metadata`) + a `poster` image for click-to-play;
  never let a below-fold player pull megabytes speculatively.
- Dimensions/aspect-ratio on the player element — same CLS contract as
  images.
- Third-party players (YouTube et al.) are the heaviest embeds on the web —
  façade them (`loading-and-code-splitting.md`).
- Streaming (HLS/DASH), adaptive bitrate ladders: decision-level here —
  involve a media pipeline when video is the product.

## Verification loop

After any media change: DevTools Network by type (sizes, formats actually
negotiated), a throttled trace for LCP phase attribution, and a CLS check
with fonts/images cold. Numbers, not vibes — the workflow in SKILL.md.
